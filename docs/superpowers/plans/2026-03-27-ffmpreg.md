# ffmpreg Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers-extended-cc:subagent-driven-development (if subagents available) or superpowers-extended-cc:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a desktop ffmpeg frontend with quality presets, waveform trimming, stream extraction, and batch processing.

**Architecture:** Electron app with React renderer, Node.js main process, and worker threads for encoding. IPC between renderer and main process. ffmpeg-static provides the bundled binary. ffprobe extracts source metadata.

**Tech Stack:** Electron 35+, React 19, TypeScript 5.9, Vite 8, ffmpeg-static, electron-store, vitest

```typescript
import { contextBridge, ipcRenderer } from 'electron';

contextBridge.exposeInMainWorld('electronAPI', {
  // Queue management
  addFiles: (paths: string[]) => ipcRenderer.invoke('queue:add-files', paths),
  removeItem: (id: string) => ipcRenderer.invoke('queue:remove', id),
  clearDone: () => ipcRenderer.invoke('queue:clear-done'),

  // Settings per queue item
  updateItemSettings: (id: string, settings: object) =>
    ipcRenderer.invoke('queue:update-settings', id, settings),

  // Encoding
  startQueue: () => ipcRenderer.invoke('queue:start'),
  cancelItem: (id: string) => ipcRenderer.invoke('queue:cancel', id),
  cancelAll: () => ipcRenderer.invoke('queue:cancel-all'),
  retryItem: (id: string) => ipcRenderer.invoke('queue:retry', id),

  // Waveform data
  getWaveform: (id: string) => ipcRenderer.invoke('file:waveform', id),

  // Progress listener
  onProgress: (callback: (id: string, percent: number) => void) => {
    ipcRenderer.on('encode:progress', (_e, id, percent) => callback(id, percent));
  },
  onStatusChange: (callback: (id: string, status: string, error?: string) => void) => {
    ipcRenderer.on('encode:status', (_e, id, status, error) => callback(id, status, error));
  },

  // Settings
  getSettings: () => ipcRenderer.invoke('settings:get'),
  updateSettings: (settings: object) => ipcRenderer.invoke('settings:update', settings),
});
```

Worker that receives `{ args: string[], outputPath: string, isStreamCopy: boolean }` via `parentPort`. Spawns ffmpeg as a child process with the args. ### Task 6: Main process queue manager + IPC handlers

**Files:**
- Modify: `electron/main.ts`

Wires together ffprobe, the queue, worker spawning, progress reporting, and all IPC handlers.

- [ ] **Step 1: Implement IPC handlers in electron/main.ts**

Handlers:
- `queue:add-files`: For each path, call `probeFile()`, create `QueueItem` with default settings (Good preset, MP4 format), return items to renderer
- `queue:remove`: Remove item from queue, cancel if converting
- `queue:clear-done`: Filter out done items
- `queue:update-settings`: Update an item's `OutputSettings`
- `queue:start`: Begin processing queue sequentially — one at a time
- `queue:cancel`: Kill worker, delete partial output, mark cancelled
- `queue:cancel-all`: Cancel current + clear queued
- `queue:retry`: Reset item to queued status, re-add to processing queue

- [ ] **Step 2: Implement queue processor**

Sequential execution loop:
1. Find first item with status `queued`
2. Call `resolveOutputPath()` to determine output
3. Call `buildFfmpegArgs()` to get CLI args
4. Spawn worker thread with args
5. Listen for progress → forward to renderer via `webContents.send('encode:progress', id, percent)`
6. On worker done → update item status, send `encode:status` to renderer
7. On worker error → update item status to failed, delete partial output, advance to next
8. Loop until no queued items remain
